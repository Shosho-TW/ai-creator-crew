# title-first

先決定標題與縮圖，再做能兌現它的內容。這個 skill 做第一步：吃一份 brief（一個題目／一句話），還沒有文章也能出標題。

## 輸入（brief，缺就問，不猜）

- **題目**：一句話講這支影片／文章在講什麼（topic-picker 選好的題目可直接貼）。
- **受眾**：給誰看？他們平常怎麼稱呼這件事？
- **角度傾向**（選填）：想主打知識、實驗、還是故事？

只給題目沒給受眾 → 先問受眾（受眾的語言決定標題會不會被點）。

## 流程

**1. 定 hook 核心.** 從 brief 抓出這支內容最值得被點進來的那個點——一個承諾、一個反差、或一個好奇缺口。這是所有標題的錨。

**2. 依 7 角度發想.** 每個角度出 2–3 條，同角度不同切入：

| # | 角度 | 例 |
|---|---|---|
| 1 | 數字清單 | 「3 個你沒聽過的…」 |
| 2 | 疑問／好奇缺口 | 「為什麼 X？」「X 嗎？」 |
| 3 | 反直覺／逆轉 | 「不是 X，而是 Y」 |
| 4 | 權威／研究錨定 | 「最新研究說…」 |
| 5 | 成本／風險重框 | 「X 個你忽略的代價」 |
| 6 | 時間／年齡限制 | 「Y 歲後…」「Y 天內…」 |
| 7 | 反直覺具體數字 | 「5g 改變…」「12% 改善」 |

**3. 收斂成 Top 3 A/B.** 從全部候選挑 3 條，必須主打 3 個不同角度（不是同一角度的三種變體），讓 A/B 測的是三個獨立的槓桿。附一句各自賭什麼情緒／承諾。

**4. 落檔.** 掛載資料夾裡有 `librarian.config.md` 或 `KB/` 結構 → 寫成 `Planning/titles-YYYY-MM-DD-<題目>.md`（`Planning/` 不存在就建立）；同日已有同題的 `Planning/topics-*.md` 就把 Top 3 附記到該檔文末。沒有 vault → 在對話交付，並問一句要不要存檔。

## 硬邊界（違者重來）

- 繁體中文，≤80 字（手機 feed 會截，寧短勿長）。
- 全形標點（：，！？），數字用 ASCII（3、65、5g）。無 emoji。
- 不寫兌現不了的 clickbait：標題承諾的，內容要做得到。發現某條標題會逼內容造假，標記出來、降級或棄用。
- 每條主打一個角度；Top 3 三個角度互不重複。
- 拍板永遠是人：這裡出候選，選哪條、改成什麼，你決定。
- vault 內只寫 `Planning/`，其他資料夾一律不碰。
